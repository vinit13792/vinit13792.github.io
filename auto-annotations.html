Case Study 01

Auto-Annotation Pipeline

Mask R-CNN PyTorch Active Learning

Phase 01

Orchestrating the
Ground Truth

The primary challenge in micro-organism detection is Label Noise. Traditional bounding boxes often fail in high-confluence microscopy where organisms overlap or exhibit stochastic boundaries. To mitigate this, we implemented a sophisticated data engineering pipeline centered on expert-verified Instance Segmentation.

I. Infrastructure & Security 🔒

We deployed a Dockerized CVAT (Computer Vision Annotation Tool) instance on a local workstation cluster. By hosting on-premise, we eliminated the latency overhead of 4K microscopy image uploads and ensured strict Data Sovereignty. The server was optimized with Nginx for concurrent access by multiple lab technicians, ensuring zero-lag navigation through multi-gigabyte datasets.

II. Annotation Protocol & IAA 📏

To ensure high Inter-Annotator Agreement (IAA), we established a strict "Polygon-First" protocol. Lab technicians were trained to handle Occlusion by inferring the underlying boundary of overlapping cells. We used a double-blind review process for the initial 5% of the dataset, calculating Cohen's Kappa to ensure labeling consistency across the 15 species before scaling the operation.

III. ETL & COCO Schema Mapping 🛠️

I engineered a custom Python ETL pipeline to transform CVAT’s raw XML/JSON outputs into the COCO JSON 1.0 specification. This involved mapping class IDs to a unified ontology and calculating precise RLE (Run-Length Encoding) masks for small-scale organisms, which are essential for stabilizing the loss function during the training phase.

Phase 02

Inference &
Model Architecture

Our modeling strategy prioritized Pixel-Level Precision. We chose Mask R-CNN over standard detectors to capture the morphological nuances (cilia, vacuoles, boundary irregularities) that define different microbial species.

I. Backbone & FPN Design 🧠

We utilized a ResNet-101-FPN (Feature Pyramid Network) backbone. The FPN was critical for multi-scale detection; it allowed the model to extract high-resolution features for small micro-organisms while maintaining a global semantic context for larger colonies. We implemented RoIAlign (Region of Interest Align) to prevent the quantization errors inherent in RoIPool, ensuring our masks remained pixel-perfect.

II. Transfer Learning & Convergence ⚡

Weights were initialized using COCO Pre-training. To prevent catastrophic forgetting, we used a staged "unfreezing" strategy: first training only the Mask and Box heads, then gradually unfreezing the ResNet backbone layers with a one-cycle learning rate schedule. This ensured the model adapted to the high-frequency textures of microscopy images without losing the generalized feature extraction capabilities of the base model.

III. TensorRT & CUDA Optimization 🚀

To hit the <3 minute inference target for 1,000+ images, we optimized the PyTorch graph using TensorRT for FP16 (Half-Precision) inference. By utilizing Asynchronous CUDA Streams, we parallelized the image pre-processing (normalization and resizing) with the GPU's forward pass, effectively eliminating CPU-side bottlenecks and maximizing GPU throughput.

Phase 03

The Active Learning
Flywheel

The final phase centered on scaling the dataset to 7,500 expert-verified instances (500 per species) while maintaining an efficient Human-in-the-Loop (HITL) workflow. We moved from passive data collection to an active regime where the model navigated the unlabelled pool to find the highest informational gain.

I. The Math of Uncertainty: Shannon Entropy 🧠

We utilized Shannon Entropy (H) as our primary acquisition function. Instead of simple confidence thresholds, entropy allowed us to quantify the total uncertainty across all 15 species labels. This ensured that microbiologists only reviewed "edge cases" where the model's decision boundaries were most fragile. The entropy for a given image $x$ is defined as:

$$H(y|x) = -\sum_{i=1}^{15} p(y_i|x) \log_2 p(y_i|x)$$

When the model's output was "flat"—meaning it couldn't distinguish between species—the entropy value spiked. By sending these specific images to CVAT, we maximized the Information Gain per human annotation.

II. MLOps Infrastructure & Automation ⚙️

To support this scale, we engineered a Dockerized pipeline that functioned as a recursive engine. High-entropy predictions were automatically routed to a priority CVAT queue via a custom API middleware. This automation reduced the manual data management overhead and ensured that the INR 6,00,000 saved by avoiding annotation services from external vendors.
