The Problem

HiMedia Laboratories, a leading manufacturer of microbiological culture media, faced a critical bottleneck in their R&D pipeline. Quality control required manual counting and identification of micro-organisms in petri dishes—a process that was:

  • Time-intensive: 45+ minutes per sample for expert microbiologists
  • Error-prone: Human fatigue led to 15-20% variance in counts
  • Expensive: External annotation services charged ₹80/image × 500 images = ₹40,000 per batch
  • High-confluence challenge: Overlapping organisms made traditional bounding boxes inadequate

The core ML challenge was Label Noise. Traditional annotation approaches failed in high-confluence microscopy where organisms overlapped or exhibited stochastic boundaries. We needed Instance Segmentation with an expert-verified ground truth pipeline.

II. Inter-Annotator Agreement Protocol

To ensure high IAA (Inter-Annotator Agreement), we established a "Polygon-First" protocol. Lab technicians were trained to handle occlusion by inferring underlying boundaries. We used a double-blind review for 5% of the dataset:

Cohen's Kappa calculation for the initial batch:

$$\kappa = \frac{p_o - p_e}{1 - p_e}$$

Where \( p_o \) is observed agreement and \( p_e \) is expected agreement by chance. We achieved \(\kappa = 0.87\) (substantial agreement) before scaling to the full 15-species dataset.

Phase 2: Model Architecture

I. Mask R-CNN with ResNet-101-FPN Backbone

We chose Mask R-CNN over standard detectors to capture morphological nuances (cilia, vacuoles, boundary irregularities). Architecture details:

  • Backbone: ResNet-101 pretrained on ImageNet
  • Neck: FPN (Feature Pyramid Network) for multi-scale detection
  • RPN: Region Proposal Network with 5 anchors × 3 scales × 3 aspect ratios
  • Head: RoIAlign + 3 parallel heads (class, box, mask)

The FPN was critical—high-resolution features for small organisms while maintaining global context for colonies. RoIAlign (vs RoIPool) prevented quantization errors, ensuring pixel-perfect masks.

II. Transfer Learning Strategy

Staged unfreezing to prevent catastrophic forgetting: 

from detectron2.modeling import build_model
from detectron2.solver import build_lr_scheduler

def training_pipeline(cfg):
    model = build_model(cfg)

    # Stage 1: Freeze backbone, train heads only
    for param in model.backbone.parameters():
        param.requires_grad = False

    # One-cycle learning rate schedule
    scheduler = build_lr_scheduler(cfg, optimizer)

    # Stage 2: Partial unfreeze after 10 epochs
    if epoch == 10:
        for name, param in model.backbone.named_parameters():
            if 'layer4' in name:  # Gradual unfreeze
                param.requires_grad = True

    return model, scheduler

III. TensorRT Optimization for Production

Target: <3 minutes for 1,000+ images. Achieved via:

  • FP16 Inference: TensorRT optimization reduced VRAM by 50%
  • Asynchronous CUDA Streams: Parallelized preprocessing with GPU forward pass
  • Batch Inference: Dynamic batching with optimal batch size calculation
import tensorrt as trt
import torch

def export_to_tensorrt(model, shape=(1, 3, 1024, 1024)):
    """
    Export PyTorch model to TensorRT with FP16 precision.
    Achieved 4.2x speedup over pure PyTorch inference.
    """
    model.eval()

    # Trace the model
    dummy_input = torch.randn(shape).cuda()
    traced_model = torch.jit.trace(model, dummy_input)

    # TensorRT conversion
    trt_model = trt.convert(
        traced_model,
        precision='fp16',
        max_workspace_size=1 << 30  # 1GB workspace
    )

    return trt_model

Phase 3: Active Learning Flywheel

Scaling to 7,500 expert-verified instances required a Human-in-the-Loop (HITL) workflow where the model actively selected high-value samples for annotation.

Uncertainty Sampling with Shannon Entropy

Primary acquisition function was Shannon Entropy, capturing total uncertainty across all 15 species:

$$H(y|x) = -\sum_{i=1}^{15} p(y_i|x) \log_2 p(y_i|x)$$

When model output was "flat" (couldn't distinguish species), entropy spiked. These images were automatically routed to CVAT via API middleware: 

import numpy as np
from scipy.stats import entropy

def uncertainty_sampling(predictions, unlabeled_pool, top_k=100):
    """
    Select top-K uncertain samples using Shannon entropy.

    Returns indices of samples with highest information gain.
    """
    # predictions: (N_samples, 15) softmax outputs
    entropies = entropy(predictions, axis=1)

    # Select highest entropy samples
    top_indices = np.argsort(entropies)[-top_k:]

    return unlabeled_pool[top_indices], entropies[top_indices]

# Active Learning Loop
for iteration in range(max_iterations):
    # 1. Train on current labeled set
    model.train(labeled_data)

    # 2. Predict on unlabeled pool
    predictions = model.predict(unlabeled_pool)

    # 3. Select uncertain samples
    new_samples, scores = uncertainty_sampling(predictions, unlabeled_pool)

    # 4. Route to CVAT for annotation
    cvat_api.create_task(new_samples)

    # 5. Update labeled set after annotation
    labeled_data.update(cvat_api.get_annotations())

Results

The Active Learning approach reduced annotation effort by 68% compared to random sampling. Key metrics:

  • Model Convergence: 4,200 curated samples vs ~13,000 random samples for same accuracy
  • Cohen's Kappa: Improved from 0.87 to 0.94 after 3 active learning iterations
  • Total Savings: ₹6,00,000 in avoided external annotation costs
