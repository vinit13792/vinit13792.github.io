Case Study 05

Propensity Models for FD Products

XGBoost Market Mix Modelling Campaign Optimization A/B Testing Feature Engineering

Driving ₹520 Crores in incremental Fixed Deposit conversions through machine learning-powered propensity scoring and intelligent campaign optimization at India's largest retail NBFC.

The Problem

Business Context

Shriram Finance manages ₹2.2 Lakh Crores in Assets Under Management, with Fixed Deposits being a critical funding source. In Q4 2024, the treasury team faced a ₹5,000 Crore funding gap that needed to be filled through retail FD mobilization within 6 months.

The Challenge

  • Budget Constraints: Marketing budget limited to ₹15 Crores—every rupee needed maximum impact
  • Low Conversion Rates: Existing blanket campaigns achieved only 0.8% conversion
  • Customer Heterogeneity: 5.2 million customers with vastly different financial profiles and FD propensity
  • Competitive Pressure: Banks offering 50-75 bps higher rates, making customer retention critical
  • Channel Fragmentation: SMS, email, WhatsApp, branch calls—no unified targeting strategy

The CRO's ask was clear: "Build me a system that tells me exactly who to call, when to call, and what to offer."

Features

Feature Engineering

The predictive power of our models hinged on feature engineering. We created 247 features across 7 categories: 

from pyspark.sql import functions as F
from pyspark.sql.window import Window

class FeatureEngineer:
    def __init__(self, spark):
        self.spark = spark

    def create_rfm_features(self, df):
        """
        RFM Features - Industry standard for financial services
        Recency: Days since last FD investment
        Frequency: Number of FD investments in past 24 months
        Monetary: Total FD investments in past 24 months
        """
        window = Window.partitionBy("customer_id")

        return df.withColumn(
            "recency_days",
            F.datediff(F.current_timestamp(), F.col("last_fd_date"))
        ).withColumn(
            "frequency_24m",
            F.count("fd_id").over(window.rangeBetween(-730, 0))
        ).withColumn(
            "monetary_24m",
            F.sum("fd_amount").over(window.rangeBetween(-730, 0))
        )

    def create_liquidity_score(self, df):
        """
        Liquidity Score: Proprietary feature capturing investable surplus
        L = (Savings + FD_maturity_proceeds - EMI_obligations) / Income
        """
        return df.withColumn(
            "liquidity_score",
            (F.col("savings_balance") +
             F.col("fd_matured_90d") -
             F.col("total_emi_obligation")) / F.col("monthly_income")
        )

    def create_behavioral_features(self, df):
        """
        Behavioral features capturing customer engagement patterns
        """
        window_90d = Window.partitionBy("customer_id") \
                          .orderBy(F.col("interaction_date").desc()) \
                          .rowsBetween(-90, 0)

        return df.withColumn(
            "email_open_rate_90d",
            F.sum("email_opened").over(window_90d) / F.sum("email_sent").over(window_90d)
        ).withColumn(
            "preferred_channel",
            F.when(F.col("sms_response_rate") > F.col("email_response_rate"), "SMS")
             .when(F.col("whatsapp_response_rate") > F.col("email_response_rate"), "WhatsApp")
             .otherwise("Email")
        ).withColumn(
            "peak_engagement_hour",
            F.hour(F.col("engagement_timestamp"))  # Aggregated in post-processing
        )

    def create_temporal_features(self, df):
        """
        Temporal features capturing investment patterns and seasonality
        """
        return df.withColumn(
            "fd_investment_velocity",
            F.col("fd_count_12m") / F.greatest(F.col("fd_count_24m"), F.lit(1))
        ).withColumn(
            "is_fd_maturity_window",
            F.when(F.col("nearest_fd_maturity_days").between(0, 30), 1).otherwise(0)
        ).withColumn(
            "quarter", F.quarter(F.current_timestamp())
        ).withColumn(
            "is_quarter_end", F.when(F.col("month").isin([3, 6, 9, 12]), 1).otherwise(0)
        )

    def create_risk_adjusted_income(self, df):
        """
        Risk-adjusted income considering delinquency history
        Used for AML/KYC compliance as well
        """
        return df.withColumn(
            "effective_income",
            F.col("monthly_income") * (
                1 - F.when(F.col("dpd_bucket") > 0, 0.3).otherwise(0)
            )
        ).withColumn(
            "investment_capacity",
            F.least(F.col("effective_income") * F.lit(0.4), F.lit(5000000))
        )

Model Development

1. Propensity Model (XGBoost)

We compared multiple algorithms (Logistic Regression, Random Forest, XGBoost, LightGBM) and selected XGBoost for its superior handling of:

  • • Class imbalance (only 0.8% positive rate)
  • • Feature interactions (captures non-linear relationships)
  • • Missing values (native handling)
import xgboost as xgb
from sklearn.model_selection import StratifiedKFold
from sklearn.metrics import roc_auc_score, precision_recall_curve

class PropensityModel:
    def __init__(self):
        self.model = xgb.XGBClassifier(
            n_estimators=500,
            max_depth=6,
            learning_rate=0.01,
            subsample=0.8,
            colsample_bytree=0.8,
            scale_pos_weight=124,  # 1:124 class ratio
            eval_metric=['auc', 'logloss'],
            early_stopping_rounds=50,
            tree_method='hist'  # Faster training
        )
        self.threshold = None

    def fit(self, X_train, y_train, X_val, y_val):
        """
        Train with early stopping on validation set
        """
        self.model.fit(
            X_train, y_train,
            eval_set=[(X_val, y_val)],
            verbose=False
        )

        # Find optimal threshold using business cost function
        probs = self.model.predict_proba(X_val)[:, 1]
        self.threshold = self._optimize_threshold(y_val, probs)

        return self

    def _optimize_threshold(self, y_true, y_prob):
        """
        Optimize threshold based on business cost matrix:
        - True Positive: Revenue of ₹50,000 (avg FD ticket)
        - False Positive: Marketing cost of ₹150
        - False Negative: Opportunity cost of ₹50,000
        """
        precisions, recalls, thresholds = precision_recall_curve(y_true, y_prob)

        # Business-optimized metric
        costs = []
        for p, r in zip(precisions, recalls):
            cost = (r * 50000 - (1-p) * 150 * r / p if p > 0 else 0)
            costs.append(cost)

        optimal_idx = np.argmax(costs)
        return thresholds[optimal_idx] if optimal_idx < len(thresholds) else 0.5

    def predict_propensity(self, X):
        """
        Return calibrated probability scores
        """
        return self.model.predict_proba(X)[:, 1]

    def segment_customers(self, X):
        """
        Segment into tiers based on propensity
        Tier 1 (Hot): >0.7 probability
        Tier 2 (Warm): 0.4-0.7
        Tier 3 (Cold): 0.2-0.4
        Tier 4 (Dormant): <0.2
        """
        probs = self.predict_propensity(X)
        return np.select(
            [probs > 0.7, probs > 0.4, probs > 0.2],
            [1, 2, 3],
            default=4
        )

MMM

Market Mix Modelling

Beyond targeting, we needed to answer: Which channels deserve more budget? We implemented a Bayesian MMM to attribute conversions across SMS, Email, WhatsApp, Branch Calls, and Digital Ads. 

import pymc as pm
import arviz as az
import numpy as np

class MarketMixModel:
    def __init__(self):
        self.channels = ['sms', 'email', 'whatsapp', 'branch', 'digital']

    def build_model(self, spend_data, conversions):
        """
        Bayesian MMM with Adstock and Saturation transformations

        Model: Conversions ~ α + Σβ_i * Saturation(Adstock(Spend_i))

        Where:
        - Adstock: Captures carryover effect of advertising
        - Saturation: Diminishing returns on spend
        - β_i: Channel effectiveness coefficients
        """
        n_weeks, n_channels = spend_data.shape

        with pm.Model() as model:
            # Priors for channel effectiveness
            beta = pm.HalfNormal('beta', sigma=1, shape=n_channels)

            # Adstock parameters (carryover)
            theta = pm.Beta('theta', alpha=2, beta=2, shape=n_channels)

            # Saturation parameters
            alpha_sat = pm.Gamma('alpha_sat', alpha=3, beta=1, shape=n_channels)

            # Apply transformations
            adstock = self._adstock_transform(spend_data, theta)
            saturated = self._saturation_transform(adstock, alpha_sat)

            # Linear combination
            mu = pm.Deterministic(
                'mu',
                pm.math.dot(saturated, beta)
            )

            # Likelihood
            sigma = pm.HalfNormal('sigma', sigma=100)
            y = pm.Normal('y', mu=mu, sigma=sigma, observed=conversions)

        return model

    def _adstock_transform(self, spend, theta):
        """
        Geometric adstock: weighted sum of current and past spend
        Spend_t' = Spend_t + θ * Spend_(t-1) + θ² * Spend_(t-2) + ...
        """
        n_weeks, n_channels = spend.shape
        adstock = np.zeros_like(spend)

        for t in range(n_weeks):
            for lag in range(min(t+1, 12)):  # Max 12 weeks decay
                adstock[t] += (theta ** lag) * spend[t - lag]

        return adstock

    def _saturation_transform(self, x, alpha):
        """
        Hill function: S(x) = x^α / (x^α + K^α)
        Captures diminishing returns
        """
        K = 100000  # Half-saturation point
        return(x ** alpha) / (x ** alpha + K ** alpha)

    def optimize_budget(self, total_budget, channel_bounds):
        """
        Linear programming to maximize expected conversions
        max Σβ_i * Saturation(Spend_i)
        s.t. ΣSpend_i = Total_Budget
             lb_i ≤ Spend_i ≤ ub_i
        """
        from scipy.optimize import minimize

        def objective(spend):
            return -np.sum(self.beta * self._saturation_transform(spend, self.alpha_sat))

        constraints = [{'type': 'eq', 'fun': lambda x: np.sum(x) - total_budget}]
        bounds = [(channel_bounds[c]['lb'], channel_bounds[c]['ub']) for c in self.channels]

        result = minimize(objective, x0=[total_budget/5]*5, bounds=bounds, constraints=constraints)
        return dict(zip(self.channels, result.x))

Key Learnings

  • Feature Engineering > Model Complexity: Our proprietary Liquidity Score was more predictive than any model hyperparameter
  • Business Thresholds Matter: Optimizing for ROC-AUC vs. business cost function gave 23% better ROI
  • Channel Attribution is Hard: MMM required 6 weeks to converge; we supplemented with holdout experiments
  • Model Monitoring is Critical: We detected feature drift within 3 weeks and retrained automatically
