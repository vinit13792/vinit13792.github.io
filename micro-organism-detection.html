What It Is

A production computer vision system that detects, counts, and calculates growth rates of micro-organisms in petri dishes. This replaced a ₹1,00,000 proprietary hardware solution with a software-based approach that achieved higher accuracy (94% vs 80%) while eliminating recurring calibration costs.

The system processes high-resolution microscopy images to identify 15 different micro-organism species, track their growth over time, and generate automated quality control reports for manufacturing compliance.

The Problem

HiMedia Laboratories relied on a commercial colony counter instrument for quality control of their culture media products. Critical issues:

  • High Cost: ₹1,00,000 per unit with annual maintenance contracts
  • Limited Accuracy: 80% detection rate with frequent false positives at colony boundaries
  • Rigid Workflow: Proprietary software couldn't integrate with internal LIMS (Laboratory Information Management System)
  • Species Limitation: Only supported 5 pre-programmed organism types vs. 15+ species in production
  • Calibration Drift: Required monthly recalibration, causing 2-day production delays

2. Preprocessing Pipeline

OpenCV-based preprocessing for consistent image quality: 

import cv2
import numpy as np

def preprocess_image(image_path):
    """Preprocessing pipeline for colony detection."""
    img = cv2.imread(image_path)

    # Convert to LAB color space for better segmentation
    lab = cv2.cvtColor(img, cv2.COLOR_BGR2LAB)
    l, a, b = cv2.split(lab)

    # CLAHE (Contrast Limited Adaptive Histogram Equalization)
    clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
    l_clahe = clahe.apply(l)

    # Merge and convert back
    lab_clahe = cv2.merge([l_clahe, a, b])
    enhanced = cv2.cvtColor(lab_clahe, cv2.COLOR_LAB2BGR)

    # Denoising
    denoised = cv2.fastNlMeansDenoisingColored(enhanced, None, 10, 10, 7, 21)

    return denoised

3. Detection Pipeline

Multi-stage detection combining classical CV with deep learning:

  • Stage 1: Watershed segmentation for initial region proposals
  • Stage 2: Mask R-CNN for instance segmentation and species classification
  • Stage 3: Post-processing filters based on morphological features
import torch
from detectron2.engine import DefaultPredictor
from detectron2.config import get_cfg

def detect_colonies(image, model_path):
    """Run Mask R-CNN inference for colony detection."""
    cfg = get_cfg()
    cfg.merge_from_file("configs/mask_rcnn_R_101_FPN_3x.yaml")
    cfg.MODEL.WEIGHTS = model_path
    cfg.MODEL.ROI_HEADS.NUM_CLASSES = 15  # 15 species

    predictor = DefaultPredictor(cfg)
    outputs = predictor(image)

    # Extract instances
    instances = outputs["instances"].to("cpu")
    boxes = instances.pred_boxes.tensor.numpy()
    scores = instances.scores.numpy()
    classes = instances.pred_classes.numpy()
    masks = instances.pred_masks.numpy()

    return boxes, scores, classes, masks

4. Growth Rate Calculation

Temporal analysis of colony growth: 

def calculate_growth_rate(detections_over_time):
    """
    Calculate exponential growth rate from time-series detections.

    Growth model: N(t) = N0 * e^(kt)
    where k is the growth rate constant
    """
    import numpy as np
    from scipy.optimize import curve_fit

    def exponential_growth(t, N0, k):
        return N0 * np.exp(k * t)

    times = np.array([d['time'] for d in detections_over_time])
    counts = np.array([d['count'] for d in detections_over_time])

    # Fit exponential model
    popt, _ = curve_fit(exponential_growth, times, counts, p0=[1, 0.1])
    N0, k = popt

    # Doubling time
    doubling_time = np.log(2) / k

    return {
        'initial_count': N0,
        'growth_rate': k,
        'doubling_time_hours': doubling_time
    }

Why It Matters

Business Impact

  • Cost Savings: Eliminated ₹1,00,000 hardware cost per unit
  • Accuracy Gain: Improved from 80% to 94% (40% relative improvement)
  • Scalability: Deployed across 5 labs with centralized model updates
  • Throughput: Processed 500+ samples/day vs 200 with old system
  • Integration: Full API integration with existing LIMS workflow

✅ When to Use

  • High-volume quality control with consistent imaging conditions
  • Known, bounded set of target organisms (classification)
  • Need for API integration with existing systems
  • Cost-sensitive environments where hardware ROI is critical
  • Time-series growth analysis requirements

⛔ When to Avoid

  • Novel organism types not in training data (would need retraining)
  • Highly variable lighting/imaging conditions
  • Regulatory requirements for certified medical devices
  • Real-time processing needs (pipeline has ~3s latency)
  • Extreme colony overlap (>70% confluence)
